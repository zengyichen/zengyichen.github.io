基于 Rhino + Grasshopper 的参数化模型设计——以同济大学 120 周年校庆纪念物设计为例

曾熠辰 2354280

胡子豪 2353418

张鸣剑 2352042

一、课题背景：用技术表现文化

1.1 文化背景

2027 年是同济大学建校 120 周年。设计纪念物能够承载同济精神文化、增强师生对母校的认同感、归属感。

“同舟共济”是同济文化与精神的核心。纪念物采用船的形状，表达了这样的精神。

1.2 技术背景

参数化设计能够通过控制变量和编程的算法，快速实现设计意图，表达复杂的几何形状和结构体系。
Grasshopper 是 Rhino 软件的程序算法生产模型的插件，是当前最主流的参数化建模工具。Rhino + Grasshopper 的参数化平台凭借其可编译性、可视性、可扩展成为应用广泛的参数化设计工具[1]。

我们用技术表现文化，展现新时代智能制造的技术与精神。

二、参数化设计：原理介绍

2.1 Rhino + Grasshopper 简介

Rhinoceros 3D（本文简称 Rhino）是一套专业的三维立体模型制作软件。Grasshopper 是一款可视化编程语言，作为采用程序算法生成模型的插件在 Rhino 环境下运行[2]。

2.2 Grasshopper：一款可视化编程语言

Grasshopper 是一款可视化编程语言，用户通过在界面（canvas）上将部件（components）连接成有向图编写程序。

部件类似于编程中的函数，可以执行特定功能，并向其他部件传递数据。按照“传递数据”这一标准，可将其分为三类：一类部件只能储存并向外传递数据，二类部件只能接受数据并执行一定功能，而占比最大、功能最丰富的三类部件能够接收数据、进行运算并输出结果。

例如，绘制一个长方体，需要以下部件：

  • 一类：长、宽、高。类型皆为数字。
  • 二类：预览模型。
  • 三类：根据数字（长、宽）绘制长方形（底面）、根据长方形和数字（高）绘制长方体。

2.3 优势：与传统设计对比

在传统的设计工作流中，设计师首先需要借助草图将自己的设计想法通过点线绘图的方式图纸化输出，设计想法的生成首先要基于针对设计对象的前期调研，而后续设计过程主要来源于设计师的主观指导。

Grasshopper 数字建模平台则集成了针对设计师图解思维的完整点、线、面操作算法，建筑师可以方便的将设计中的点线绘图思维数字化，并借助 Rhino 可视化输出，使得整体设计逻辑算法化而清晰可控，同时通过编译过程中不同参数的设置，建筑师也可以方便的更改某一步设计逻辑，实现多维度的对比[3: 84]。

三、参数化设计：内容与方案

3.1 设计内容：亟待探索

纪念物由两部分组成。下方底座为船形，寓意“同舟共济”；上方为树形，代表同济大学标志性的樱花树。

我们暂未确定更具体的设计细节，且后续可能对该方案作出改动。前文已经提及，参数化设计更改参数、逻辑都十分便捷。因此，变动设计方案的时间成本极低，暂时不需要过于具体的设计方案。当前的任务主要是探索各种设计方案与相应的实现技术。

而经过研究，我们探索出这两部分的不同设计方案如下。

3.2 设计方案：手动绘制

图形等一类数据（见 2.2）可以在 Rhino 中手动绘制再导入 Grasshopper 进行运算。参数化设计与传统 CAD 建模并不冲突，而是相辅相成、互为补充的关系。

船的形状并非简单的几何体，且其造型参数等需要直接调整以达到最佳视觉效果，并不适合完全由参数化这一高度抽象的方式生成。我们计划将传统船只的造型特点进行抽象，并作出改动，在 Rhino 中手动绘制轮廓线，此后再参数化生成其结构。

3.3 设计方案：Python 辅助

Grasshopper 内置了脚本执行部件，支持的语言包括 C#、VB 和 Python。我们选用 Python 辅助设计上方樱花树部分。

选择 Python 编程语言的主要原因在于三点[3: 75]：

  • Python 代码开源的，资料丰富易于获取；
  • Python 语言是解释型编程语言，学习成本适中，易于上手；
  • Python 有非常丰富的第三方库可以使用，如专门为 Grasshopper 开发的 GhPython 库和适于简易作图的 Turtle 库等。

在程序设计中，建立树形结构的常用方法是函数递归，即调用自身。例如图 3-1 所示的二叉树中，节点 {1,2,3}\{1, 2, 3\}{1,2,3} 与 {2,4,5}\{2, 4, 5\}{2,4,5} 形状相同。利用这一“自相似性”，在绘制时只需在 1 处执行一次绘图函数，在 4 ~ 7 处依据“到达指定高度”返回，即可简洁地完成绘制。而仅仅依靠 Grasshopper 默认部件并不容易实现这一逻辑。

4.3 总结

从以上设计过程中可以看出，参数化设计的关键在于总结出特定结构的数学模型。

此外，对比两个案例可以发现，案例 2 设计过程更简单，但视觉效果更丰富，这源于 Voronoi 3D 的应用。可见，部件中常常蕴含新颖的设计原理，设计者需要同时具有“运用部件”和“设计模型结构”的能力。

我隐藏博客，想与过去的某些经历告别，但难以如愿。这几个月参加了很多面试，每次都要把一些事拿出来翻来覆去地炒作，这是一些抹不掉的东西；也有留不下的东西，一些次要却有趣的细节，失去这些之后我若向别人讲述经历，则会像神志不清的老人念叨过去一样乏味。但无论如何是结束了，新的生活已经开始。得益于竞赛经历，之后我还是愿意在计算机和数学领域探索。而即使不考虑自己经历带来的偏见，我也始终认为 OI 甚至整个算法竞赛圈的模式以及风气是病态的，如果要做出评价，我认为这是一段并不美好但能锻炼人的经历。

对于未来的规划暂时不明确，应该还是会学计算机方面的东西？最近写入党申请书，在一堆报菜名中加了点私货，大意就是网络可以实现生产资料共享，实现生产的去中心化从而向共产迈进一步。这显然是对计算机方面的技术而言。虽然这观点很扯淡但确实写出了计算机吸引我的地方。不过我所在的书院大一上的是工科通识，就是数理化信息都学，所以这里暂时也不会有很多 CS 内容。

至于这个博客，在整理笔记之外，最大的作用可能是用来发泄过剩的表达欲……如果有的话。我是希望自己一直能表达一些东西的，而且希望这其中不要有太多回忆录。

在以前博客的唯一一篇文章中，我曾经写了这句话：

而我显然不是能把“去码头整点薯条”作为目标的人，那要等到什么时候能走出来呢？可能那天还会回来写个 blog。

不是这一篇。我仍然在等着那一天。
